[기본 문법]
$.ajax({
	type : "GET/POST", (ajax 요청시 사용할 http 메서드 결정)
	url : "주소", (ajax 요청할 주소 url)
	data : "값", (ajax 요청시 전달할 데이터 값, 파라메터)
	dataType : "응답 데이터의 타입 (html,text,json)",
	sucess : function(data,status,jqXHR){
		(ajax 요청이 성공적으로 처리된 경우 실행)
		(data - 정상 처리의 결과)
		(status - 리턴 결과 문자)
		(jqXHR - xml, html, request 객체 - httpStatus 값)
	},
	error : function(){
		ajax 요청이 실패한 경우 실행
	}
});

JSON(Javascript Object Notation) 타입
=> 자바 스크립트 객체 표현식

- 개방형 표준 데이터 포멧 (거의 모든곳에서 해당 타입의 데이터 사용가능)
- key : value 쌍으로 정보를 저장 => 데이터 저장 크기가 적음 => 전송 속도가 빠르다
  xml : <tag>key</tag> , <tag>value</tag>
- ajax 응답 결과가 다른 도메인 주소라도 요청 처리가능
- 주로 웹 애플리케이션에서 사용
- 대부분의 플랫폼에서 JSON 타입 지원/ 브라우저에도 JSON Parser 포함

단점...
- 자바스크립트 문법을 따름 => 예민함(엄격함)
- 자바스크립트 코드를 포함 가능(블법적인 코드도 숨길 수 있음)

	[ 문법 ]
	- 속성(key) : 값(value) 쌍으로 저장
	- 속성은 "" 로 묶어서 표기
	- 속성의 갯수가 여러개일때 , 로 구분
	- 마지막 요소는 , 없음
	
	- JSON 객체 중괄호 {} 로 표기
	- JSON 배열 대괄호 [] 로 표기
	
	- 값은 데이터 타입에 따라서 표기법 변경
	- 정수 : 단, 2진수, 8진수, 16진수는 표현 X
	- 실수 : 1) 고정 소수점 : 3.14
			2) 부동 소수점 : 1e4, 6.7E-5
	- boolean 제공 : true/false
	- null 표현 O
	- 대부분의 데이터가 문자열(String) : ""로 묶여있는 0개 이상(공백 문자 포함이라서) 유니코드 문자, 이스케이프 문법 지원
	- 배열
	
	- 날짜정보 처리시 주의
	1) 정수 타입의 날짜 정보(ms)
	2) 문자 타입의 날짜 정보
